Programming Languages:
The Early Development


    First Generation Languages

      Earlier computers, without the help of a stored program, were programmed by modifying their circuitry. Programs were later written in machine code, which is a set of instruction, in binary or decimal, the hardware can excute directly. Those languages were later re-named to First-generation programming languages.

    Second Generation Languages

      Second generation programming languages or assembly languages were the next step in development, They were still closely tied to the instruction set architecture of the specific computer, Second generation languages were intended to make the program more readable by humans and relived the programmer of errors previously faced.

    Third Generation Languages

      The first High-level programming language or Third generation programming languages were written in the 1950s. An example is Plankalkul developed for the German Z3 between 1943 and 1945. However not implemented until the late 90s

      Short Code

        Proposed in 1949 by John Mauchly, Short code was one of the first high-level languages ever developed for an electronic computer. Short code, unlike Machine code, represented mathematical statements in an understandable form. However, since the code could not be read directly by the hardware it made the excution process much slower than that of the machine code

      Autocode

        Developed by Alick Glennie at the University of Manchester as a porgramming language, it used a compiler to auto-convert the language into machine code.
        The first code and compiler were developed for the Mark 1 at the University of Manchester in 1952 and is considered to be the first compiled high-level language
